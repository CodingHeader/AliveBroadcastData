## 四、油猴脚本使用

### 安装
1. 浏览器安装 [Tampermonkey](https://www.tampermonkey.net/) 扩展
2. 新建脚本，将 `tampermonkey/alive-broadcast-sync.user.js` 内容粘贴进去
3. 保存，脚本会自动在 `life.douyin.com` 域名下运行

### 手动触发采集
打开抖音来客「线索大屏」页面，按 `F12` 打开控制台，输入：
```js
window.startCollection()
```
按回车执行，脚本将自动遍历并采集前N天的全部直播场次数据（N由COLLECT_DAYS配置，默认1=昨天）。

> **注意**：手动触发不消耗自动采集配额（`today_attempts`），两者相互独立。手动采集成功后同样会标记 `last_collect_date`，当天不再自动触发。

### 配置项（脚本顶部 CONFIG 区域）
| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `SERVER_URL` | `http://36.134.158.50:12306` | 服务端地址 |
| `CHECK_INTERVAL` | `60000` | 心跳检测间隔（毫秒） |
| `COLLECT_HOUR` | `0` | 自动采集触发小时（0=凌晨0点） |
| `COLLECT_DAYS` | `1` | 采集天数: 1=昨天, 2=昨天+前天, 建议上限7 |
| `RETRY_COUNT` | `3` | 发送失败重试次数 |
| `RETRY_DELAY` | `5000` | 重试间隔（毫秒） |
| `MAX_DAILY_ATTEMPTS` | `3` | 每天自动采集最大尝试次数（含失败） |

## 五、脚本运行逻辑详解

### 5.1 不执行 `window.startCollection()` 时的运行逻辑

脚本加载后进入「主入口」（第826-835行），分两种情况：

**情况A：页面是自动刷新后重启的**
- 脚本检测到 `pending_collect === 'true'`
- 清除标记，3秒后**自动执行** `startCollection()`
- 此情况出现在：心跳检测触发自动刷新后

**情况B：正常页面加载**
- `pending_collect === 'false'`
- 启动 `startHeartbeat()` 心跳检测
- 心跳每 `60秒` 执行一次检查：
  - 条件1：当前小时 >= `COLLECT_HOUR`（默认0点，即全天满足）
  - 条件2：`last_collect_date`（上次采集日期）≠ 今天
  - **两个条件同时满足** → 设置 `pending_collect='true'` → `location.reload()` 刷新页面
  - 任一条件不满足 → 继续等待下一次检测

**总结**：不手动触发时，脚本只做一件事——每60秒检查一次"今天是否已经自动采集过"，如果还没采集就刷新页面自动开始。

### 5.2 自动刷新机制与"每日重试3次"策略

**设计目标**：每天0点自动采集前一天的直播数据；采集完成后当天不再采集；若采集失败，最多自动重试3次，3次后彻底放弃，等待第二天0点。

**实现机制**：新增 `today_attempts` 计数器（基于 `last_attempt_date` + `today_attempts` 两个GM存储值）。

**具体触发链**：

| 场景 | 行为 |
|------|------|
| **首次触发（0点后）** | `today_attempts = 0` → 自动+1 → 执行采集（第1次尝试） |
| **采集成功** | `last_collect_date = today`，心跳检测当天不再触发 |
| **采集失败/异常** | `today_attempts` 已记录为1，心跳继续检测 → 页面刷新 → 再次尝试（第2次） |
| **第2次仍失败** | `today_attempts = 2`，心跳继续触发 → 再次尝试（第3次） |
| **第3次后（无论成败）** | `today_attempts = 3`，`startCollection()` 开头直接跳过，心跳不再触发刷新，当天彻底静默 |
| **第二天0点后** | `last_attempt_date` 是昨天 → `today_attempts` 自动重置为0 → 重新开始 |

**关键点**：
- `startCollection(isAuto)` 区分自动/手动触发：仅自动触发（`isAuto=true`）消耗 `today_attempts` 配额，手动触发（`isAuto=false`）不受限
- `startCollection()` **一进入就记录尝试次数**（仅自动触发时），即使后续异常退出，次数也已计入
- 心跳触发条件：当前小时 >= 0 且 `lastCollect !== today` 且 `todayAttempts < 3`
- 3次自动尝试耗尽后，发送桌面通知 + 服务端告警，页面**不再自动刷新**，直到第二天0点
- `last_collect_date` 记录的是**采集发起日期**（缓存 `collectDay`），而非结束日期，防止跨午夜边界问题
- 心跳 interval 在当天无需再触发时自动 `clearInterval` 停止

### 5.3 刷新频率

| 参数 | 值 |
|------|-----|
| 心跳检测间隔 | **60秒**（`CONFIG.CHECK_INTERVAL = 60 * 1000`） |
| 自动触发条件 | 当前小时 >= 0 且 `lastCollect !== today` 且 `todayAttempts < 3` |
| 页面刷新后延迟执行采集 | **3秒**（`setTimeout(() => startCollection(), 3000)`） |
| 场次切换等待 | 0.5秒 ~ 30秒不等（取决于数据加载） |

**当天尝试次数耗尽后（today_attempts >= 3）**：心跳检测判定条件不满足，**页面不再自动刷新**，保持静止直到第二天0点。

**最坏情况（有重试上限）**：即使采集连续失败，最多也只会在60秒内刷新3次，之后彻底停止。

### 5.4 如何避免非预期刷新

| 方法 | 操作 |
|------|------|
| 临时关闭 | 在 Tampermonkey 面板中关闭此脚本 |
| 重置今日自动尝试次数 | 控制台执行 `GM_setValue('today_attempts', '0')` |
| 重置采集日期 | 控制台执行 `GM_setValue('last_collect_date', '')` |
| 同时重置两者 | 控制台执行 `GM_setValue('today_attempts', '0'); GM_setValue('last_collect_date', '')` |
| 手动触发不受配额限制 | 直接执行 `window.startCollection()` 即可，不消耗自动配额 |
| 修复根因 | 检查 `startCollection()` 失败原因（控制台查看日志 `[直播同步]` 前缀的报错） |

## 六、常见问题

**Q1：执行 `window.startCollection()` 后控制台没有反应？**
> 确保当前页面是 `life.douyin.com` 域名的「线索大屏」页面，且页面已完全加载。

**Q2：采集数据发送失败？**
> 检查 `SERVER_URL` 是否可访问，以及服务端是否已启动。可在浏览器直接访问 `${SERVER_URL}/api/sessions` 测试连通性。

**Q3：如何查看采集进度？**
> 打开浏览器控制台（F12），过滤 `[直播同步]` 前缀的日志，可看到每场次的采集、发送、跳过状态。

**Q4：如何重置本地缓存的待发送数据？**
> 控制台执行：`GM_setValue('pending_data', '[]')`

**Q5：数据库文件在哪里？如何重置？**
> 数据库文件位于 `server/data.db`。删除此文件后重启服务，数据库会重新初始化（注意：所有数据将丢失）。

## 八、核心功能说明

### 8.1 主播体系

**功能说明：**
- 后台可配置主播列表（名称、头像等）
- 场次详情页支持关联多个主播（管理员可见）
- 数据看板显示主播表现Top3（按成单率排序）
- 趋势分析支持按主播筛选数据
- AI分析报告自动包含本场主播信息

**数据流向：**
```
后台配置主播 → 场次关联主播 → 统计聚合 → 前台展示
```

### 8.2 成单闭环

**功能说明：**
- 线索列表可直接标记成单（金额/日期/员工）
- 成单列表显示关联线索信息
- 防止同一线索重复成单（数据库UNIQUE约束）
- 主播统计包含成单数据（成单数/成单金额/成单率）
- 日报/周报/月报包含成单统计

**数据流向：**
```
线索 → 标记成单 → Deal表 → 统计聚合 → 报告生成
```

### 8.3 告警逻辑

**告警类型：**
1. **零留资告警**：最新场次零留资且已结束≥2小时（避免数据同步延迟误报）
2. **线索成本异常**：最新场次线索成本高于近N天均值50%

**频率限制：**
- 同类型告警10分钟内只触发一次（内存级别，进程重启重置）
- 避免告警轰炸

### 8.4 报告中心

**报告类型：**
- **场次分析**：单场AI分析报告（调用OpenAI API）
- **日报**：指定日期的汇总报告（含成单统计）
- **周报**：指定周期的汇总报告（格式：2026-W20）
- **月报**：指定月份的汇总报告（格式：2026-05）

**操作支持：**
- 手动生成（场次/日期/周期/月份）
- 下载Markdown文件
- 重新生成（更新内容）
- 删除报告

**数据驱动设计：**
- 场次分析Tab：先展示所有场次，未生成报告的显示"生成AI分析"按钮
- 日报Tab：先展示所有有场次数据的日期，未生成报告的显示"生成日报"按钮

## 九、故障排查

**Q6：启动时报错 `ModuleNotFoundError: No module named 'xxx'`？**
> 检查 `requirements.txt` 是否完整，执行 `pip install -r requirements.txt` 安装依赖。

**Q7：数据库迁移脚本执行失败？**
> 检查 `server/data.db` 文件权限，确保可读写。如遇重复数据错误，手动清理 `deals` 表中重复的 `lead_id` 后再执行。

**Q8：AI分析报告生成失败？**
> 检查后台AI配置（API Key、Base URL、Model），确保OpenAI格式API可访问。查看服务端日志确认具体错误。

**Q9：邮件推送未收到？**
> 检查后台邮箱配置（SMTP地址、端口、发件人、密码、收件人）。测试时可在控制台手动触发 `scheduler.py` 中的定时任务。

**Q10：油猴脚本自动采集未触发？**
> 检查脚本是否启用、`COLLECT_HOUR` 配置是否正确、页面是否在 `life.douyin.com` 域名。查看控制台 `[直播同步]` 日志确认心跳检测状态。
