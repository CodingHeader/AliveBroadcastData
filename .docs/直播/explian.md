# 问题1

如果我有两个直播间，该返回的数据能分得清是来自哪个直播间的线索吗.

可以明确地告诉您：**能分清**。但具体通过哪个字段来区分，取决于您所说的“两个直播间”在实际业务中是指哪种情况。

需要先澄清一个平台规则：**同一个抖音账号在同一时间只能开启一个直播间**。因此，您提到的“两个直播间”通常是指以下四种业务场景之一。针对每种场景，抖音开放平台的线索 API 和 Webhook 提供了不同的字段来帮您精准区分：

### 场景一：同一个账号，在不同时间段开的两场直播（例如：上午一场，晚上一场）
如果您是指同一个抖音号的不同直播“场次”，区分方式如下：
*   **核心区分字段：`create_time`（留资时间）**。这是最准确的物理区分方式。您只需在系统中记录每场直播的“开播时间”和“下播时间”，根据线索的 `create_time` 即可精准归因到具体的直播场次。
*   **辅助区分字段：`live_id` / `room_id`（直播间场次 ID）**。部分线索详情接口会直接返回具体的直播间场次 ID。如果接口返回了此字段，您可以直接通过 `live_id` 来区分是哪一场。

### 场景二：同一场直播中，挂载了两个不同的留资商品或组件（例如：小风车里挂了“9.9元体验券”和“19.9元套餐”）
如果您是在同一场直播中，通过挂载不同的团购商品或留资卡片来收集线索：
*   **核心区分字段：`item_id`（商品 ID）或 `product_id`**。当用户点击小风车里的不同商品留资时，API 返回的数据中会带上该商品的唯一 ID。
*   **辅助区分字段：`component_id`（组件 ID）**。如果您使用的是自定义的留资组件（如高级在线预约），返回数据中会包含组件 ID。
*   **开发建议**：在您的 CRM 系统中，将 `item_id` 映射为具体的商品名称（如“体验券A”、“套餐B”），这样销售看到线索时，立刻就知道用户是对哪个产品感兴趣。

### 场景三：您其实有两个不同的抖音账号（矩阵号）分别开播
如果您是指公司有两个企业号（比如“品牌主账号”和“门店子账号”）各自开播：
*   **核心区分字段：`anchor_id`（达人/账号 ID）或 `poi_id`（门店 ID）**。
    *   如果两个账号绑定了不同的门店，直接通过 `poi_id` 区分。
    *   如果绑定同一个门店但账号不同，通过 `anchor_id` 区分是哪个抖音号带来的线索。

### 场景四：同一场直播，但投放了两个不同的广告计划（巨量本地推）
如果您是通过付费投流，在同一个直播间建了两个不同的广告计划（比如计划A投“人群包”，计划B投“达人相似”）：
*   **核心区分字段：`ad_id`（广告计划 ID）或 `campaign_id`**。
*   线索数据中会携带流量来源的广告计划 ID。您可以直接将 API 返回的 `ad_id` 与巨量本地推后台的计划 ID 进行比对，从而计算出每个广告计划的 ROI 和线索成本。


### 💡 给您的开发与业务落地建议

为了确保您的系统能完美区分这些线索，建议在开发时做好以下两步：

#### 1. 数据库设计要“留足字段”
在您的数据库（或抖音来客对接的中间表）中，除了基础的手机号、姓名外，**务必把以下字段都建好并落库**，哪怕现在不用，以后做数据分析时也会非常有用：
*   `clue_source` (大场景：直播/短视频/门店)
*   `poi_id` (门店 ID)
*   `item_id` (商品/团购 ID)
*   `ad_id` (广告计划 ID)
*   `live_id` (直播场次 ID)
*   `create_time` (精确到秒的留资时间)

#### 2. 业务上的“物理防混淆”手段（兜底方案）
如果由于接口版本原因，某些极细粒度的字段（如 `live_id`）没有返回，您可以通过业务手段来辅助区分：
*   **差异化钩子**：在两个不同的直播间（或两场直播），主播口播引导用户领取**不同的优惠券**或**填写不同的备注**。例如：“A直播间的宝宝扣1，B直播间的宝宝扣2”。用户留资后，销售一看备注或领取的券码，就知道是哪个直播间来的。
*   **专属客服/分机号**：如果条件允许，不同直播间配置不同的企业号私信自动回复话术，或者在留资卡片上留下不同的分机号/分机后缀。

**总结**：只要您的直播间正确挂载了组件，抖音开放平台返回的线索数据中一定包含了足够的维度（时间、商品、账号、广告计划等）让您区分来源。您只需要在解析 JSON 数据时，把这些标识字段提取出来并存入您的业务系统即可。
