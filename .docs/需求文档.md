# 抖音直播数据分析系统 — 需求文档

## 一、项目概述

### 1.1 项目名称
AliveBroadcastData（抖音直播数据分析系统）

### 1.2 业务背景
公司通过抖音直播进行线上引流（学历提升业务），获取潜在客户线索（手机号/表单/私信），再由员工通过电话等方式跟进线索，最终促成付款成单。当前数据分散在"抖音来客"平台，缺乏自动化采集和系统化分析能力。

### 1.3 项目目标
- 自动化采集抖音来客"线索大屏"页面的每场直播数据
- 存储到服务端SQLite，提供数据看板和AI分析报告
- 支持邮件推送、报告下载、成单标记、主播关联分析
- 打通"直播引流 → 线索获取 → 员工跟进 → 成单"全链路

### 1.4 核心用户

| 角色 | 使用场景 |
|------|----------|
| 运营人员 | 查看前台看板、下载报告、了解每场直播效果 |
| 管理层/领导 | 接收邮件推送、查看AI分析报告 |
| 管理员 | 后台配置主播、标记成单、配置AI和邮箱参数 |

## 二、功能需求

### 2.1 油猴脚本（数据采集端）

#### 2.1.1 基本信息
- 运行环境：Tampermonkey，运行在 `life.douyin.com` 线索大屏页面
- 触发方式：每天0点自动执行（浏览器24小时保持打开）
- 可配置项：服务器地址（IP:Port），写在脚本顶部常量

**@grant 权限清单：**
```
@grant GM_xmlhttpRequest  — 跨域POST数据到服务器
@grant GM_setValue        — 存储上次采集时间，防重复
@grant GM_getValue        — 读取上次采集时间
@grant GM_notification    — 采集完成桌面通知（可选）
```

**异常恢复策略：**

| 异常场景 | 恢复策略 |
|----------|----------|
| 浏览器标签页休眠 | `setInterval` 每分钟心跳检测，恢复后重新检查时间 |
| 网络断开 | POST失败重试3次，间隔5秒；全部失败则 `GM_setValue` 存入本地缓存，下次执行时补发 |
| 页面加载失败 | 刷新页面重试，最多3次，仍失败则跳过并记录日志 |
| 浏览器崩溃/重启 | 依赖 `GM_getValue` 持久化的上次采集时间，重启后自动判断是否需要补采 |
| API返回错误 | 记录错误日志，跳过当前场次继续下一场 |

#### 2.1.2 场次遍历逻辑
1. 刷新页面
2. 根据 `CONFIG.COLLECT_DAYS` 生成采集日期列表（1=昨天，2=昨天+前天，N=最近N天）
3. 对每个采集日期：
   a. 读取当前场次的开播时间（格式："05-21 09：28～05-21 13：36"，以"～"前的开播时间判断日期）
   b. 如果开播日期是今天或目标日期之后 → 点"上一场"继续往前
   c. 如果开播日期是目标日期之前 → 点"下一场"进入目标日期第一场
   d. 开始采集当天所有场次数据
   e. 采集完一场 → 点"下一场"
   f. 结束条件：
      - "下一场"按钮变灰（`cursor-not-allowed`）
      - 或下一场的开播日期不在采集日期范围内
4. 全部日期采集完成后标记 `last_collect_date`

#### 2.1.3 按钮定位方式
- 上一场：`svg[data-log-name="上一场"]`
- 下一场：`svg[data-log-name="下一场"]`
- 变灰判断：SVG的class包含 `cursor-not-allowed` = 禁用；`cursor-pointer` = 可点击

#### 2.1.4 每场采集6类数据
| # | 数据模块 | 说明 |
|---|----------|------|
| 1 | 核心指标 | 44个字段（曝光、观看、互动、留资、消耗等） |
| 2 | 复盘表 | 指标/场次表格数据 |
| 3 | 线索分析列表 | 每条线索：时间、昵称、ID、电话(脱敏)、城市、路径、标签、投放账户 |
| 4 | 评论明细 | 昵称、是否留资、评论内容、评论时间 |
| 5 | 高意向用户 | 昵称、头像URL、评论数、停留时长、状态 |
| 6 | 图片资源 | 用户头像等img标签的src URL |

#### 2.1.5 DOM定位策略
- 优先使用 `data-log-name`、文本内容、结构位置定位元素
- 不依赖动态class名（如 `V7dU-xxx`）
- 接受抖音来客改版时需要维护更新的风险

#### 2.1.6 防重复机制
- 发送前调用 `GET /api/session/check?start_time=xxx` 检查是否已入库
- 已入库则跳过该场

#### 2.1.7 扩展预留
- 推送频率可能改为每小时（采集最新已结束的场次）
- 脚本触发间隔需可配置化

### 2.2 服务端（数据接收+存储+分析）

#### 2.2.1 数据接收API
- 接收油猴脚本POST的单场完整数据
- 解析并存入SQLite各表
- 返回成功/失败状态

#### 2.2.2 前台看板（公开访问，无需登录）
- **数据看板首页**：今日/近7日/近30日核心KPI卡片、趋势图、最近场次列表
- **场次详情页**：单场完整数据展示 + AI分析报告 + 关联主播
- **报告中心**：按类型（场次/日/周/月）筛选、预览、下载
- **趋势分析**：多场对比、主播对比、时间趋势图表

#### 2.2.3 后台管理（需登录）
- **登录**：账号密码（默认 admin/admin123），单用户，支持多点登录
- **成单管理**：手动标记哪些客户已付款成单，关联场次和线索
- **主播配置**：增删主播，为每场直播关联1个或多个主播
- **AI配置**：API_KEY、base_url、model_id（兼容OpenAI格式）、系统提示词、用户提示词
- **邮箱配置**：SMTP信息、多个收件人邮箱
- **系统设置**：修改管理员密码

#### 2.2.4 AI分析
- 触发方式：每小时自动检查未分析场次 + 手动点击
- 输入：每场直播的全量数据（指标+线索+评论+高意向用户）
- 输出：Markdown格式分析报告
- 存储：SQLite reports表
- 配置：系统提示词和用户提示词后台可配置，配置后设为默认

#### 2.2.5 邮件推送
- SMTP：smtp.163.com:465（SSL）
- 发件人：后台配置
- 收件人：支持多个邮箱（后台配置）
- 推送时机：与数据同步后自动推送（当前每天0点后，未来可能改为每小时）
- 邮件格式：HTML邮件（大方有条理）
- 邮件内容：昨天总数据 + 每场场次数据 + AI分析结果

#### 2.2.6 报告生成
- 类型：单场报告、日报、周报、月报
- 周报：每周一自动生成上周数据
- 月报：每月1号自动生成上月数据
- 日报：每天自动生成前一天数据
- 下载：用户通过筛选按钮筛选后下载

#### 2.2.7 定时任务
| 任务 | 频率 | 逻辑 |
|------|------|------|
| AI分析 | 每小时 | 查询未分析场次，逐场调用AI，存储报告 |
| 邮件推送 | 每天（数据同步后） | 推送前日汇总+每场数据+AI分析 |
| 日报生成 | 每天 | 聚合前一天所有场次 |
| 周报生成 | 每周一 | 聚合上周数据 |
| 月报生成 | 每月1号 | 聚合上月数据 |

## 四、数据来源字段清单

### 4.1 核心指标（44字段，来自线索大屏）

| 分类 | 字段列表 |
|------|----------|
| 流量 | 直播间曝光次数、曝光进入率、直播间观看次数、累计观看人数、粉丝占比、>1分钟观看次数 |
| 留存 | 人均观看时长、粉丝停留时长、最高在线人数、平均在线人数、实时在线人数 |

| 成交 | 直播间成交金额、营销订单数、订单人数、订单成本 |
| 投放 | 营销消耗 |
| 其他 | 打赏次数/金额 |

留资时间、用户昵称、线索ID、电话(脱敏)、商品名称、所在城市、留资路径、特征标签、来源投放账户

### 4.3 评论明细字段
昵称、是否留资、评论内容、评论时间

### 4.4 高意向用户字段
昵称、头像URL、评论数、停留时长、状态（已留资/未留资/忽略）

## 五、业务规则

### 5.1 付费/自然线索区分
- leads.ad_account 不为空且不为"--" → 付费线索
- leads.ad_account 为空或为"--" → 自然线索
- 统计时按此规则分类

### 5.2 一场多主播归属
- 一场直播可关联多个主播（通过session_anchors表）
- 主播维度统计时，该场数据同时计入所有关联主播（不拆分）
- 即"主播A参与了10场"而非"主播A独占了10场"

### 5.3 成单归属
- 成单关联到客户留资的那场直播（deals.session_id）
- 一个客户只能成单一次（同一lead_id不重复标记）

### 5.4 线索有效性
- NULL = 未验证（默认）
- true = 有效（员工确认可联系）
- false = 无效（空号/拒接/非目标客户）

### 5.5 报告中ROI计算
- ROI = 成单总金额 / 营销总消耗
- 线索→成单转化率 = 成单数 / 总线索数
- 仅在周报/月报中展示（单场报告不含，因成单有滞后性）

## 六、系统限制

| 限制 | 说明 |
|------|------|
| 不支持实时监控 | 本系统分析已结束的直播场次，无法监控进行中的直播（抖音来客平台限制） |
| 电话不可逆 | 存储脱敏格式，无法还原完整号码 |
| 依赖浏览器在线 | 油猴脚本需浏览器24小时运行，无守护进程 |
| 单账号 | 仅采集当前登录的抖音来客账号数据 |
